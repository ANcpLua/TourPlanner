# Policy

- AI may be used as a pair-programming and review assistant.
- Every submitted change must be understood by the student who submits it.
- Students must be able to explain what changed, why it changed, and which project owns the responsibility.
- Copy-paste without understanding is not acceptable.
- AI output must be reviewed before it is accepted.
- Generated code must follow the project architecture and ownership rules.
- Students remain responsible for correctness, build status, test status, and code quality.
- Secrets must not be placed in client code, prompts, commits, or screenshots.
- If AI proposes a change that violates ownership boundaries, the boundary rule wins.
- If AI proposes a shortcut that hides a design problem, the design problem must be fixed instead.
